### Ready to Code?

If you'd like to write some code, here is a simple workflow to get you started.

1.  **Find Something to Work On:**
    *   Look for issues labeled [`good first issue`](https://github.com/glozinc/Dict_Whisperer/labels/good%20first%20issue) ‚Äì these are specifically chosen for new contributors.
    *   Tackle any open bug or feature request that interests you.
    *   Work on something from the [Roadmap](README.md#roadmap-ideas).

2.  **Set Up Your Environment:**
    *   **Fork** the repository to your own GitHub account.
    *   **Clone** your fork to your computer: `git clone https://github.com/YOUR_USERNAME/Dict_Whisperer.git`
    *   **Navigate** into the project folder: `cd Dict_Whisperer`
    *   **Create and activate** a Python 3.13 virtual environment (see the `README.md` for detailed steps).
    *   **Install in editable mode:** This is a special command that lets you test your code changes live.
        ```bash
        pip install -e .
        ```

3.  **Write Your Code:**
    *   Create a new branch for your changes: `git checkout -b my-awesome-feature`
    *   Write your code and test it to make sure it works.
    *   Don't worry about perfect code style. Just do your best! We can clean it up together.

4.  **Submit Your Contribution (Create a Pull Request):**
    *   Commit your changes with a simple message: `git commit -m "feat: Add my awesome feature"`
    *   Push your branch to your fork: `git push origin my-awesome-feature`
    *   Go to the original DictWhisperer repository on GitHub and open a **Pull Request**.
    *   In the description, just briefly explain **what** you changed and **why**.
